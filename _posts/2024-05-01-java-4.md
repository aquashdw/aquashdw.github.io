# 연산자 (Operators)

변수는 데이터를 저장한다. 그리고 여러 데이터를 모으고 조작하여 정보를 만든다. 그리고 이런 정보를
주고받아 사용자에게 유의미한 결과를 제공하는 것이 프로그램이다. 예를 들어 식당의 손님이 주문한 메뉴를 입력받아
해당 주문의 총 금액을 돌려주는 기능을 가진 프로그램을 생각해볼 수 있다. 사용자는 사전에 어떤 메뉴가 어떤 가격인지를
설정해두고, 고객의 주문을 입력받아 총 금액을 출력하는 방식으로 코드를 작성하는 것이다.

이처럼 데이터를 다루는 가장 간단한 방법은 계산이다. 그리고 이런 계산을 할때 사용하는 것이 연산자이다.

![연산자와 피연산자](/assets/images/java/4/4-1.png)

음....그냥 수식 쓰듯이 사용하면 되기는 하다. 물론 단순히 계산기처럼 쓰는건 아니고 단순히 계산을 하는 연산자 외에 다른 용도의 연산자도 있다.
사칙연산을 할때 사용하는 연산자 역시 Java에서 쓸 때는 몇가지 알아둬야 할 부분도 있으니 살펴보자.

## 산술 연산자

흔한 계산을 위한 연산자이다. 여기에 나머지를 구하는 모듈러 연산자까지 있다.

| 연산자 | 설명                                       |
|-----|------------------------------------------|
| `+` | 왼쪽의 피연산자에서 오른쪽 피연산자를 더한 값을 반환한다.         |
| `-` | 왼쪽의 피연산자에서 오른쪽 피연산자를 뺀 값을 반환한다.          |
| `*` | 왼쪽의 피연산자에서 오른쪽 피연산자를 곱한 값을 반환한다.         |
| `/` | 왼쪽의 피연산자에서 오른쪽 피연산자를 나눈 값을 반환한다.         |
| `%` | 왼쪽의 피연산자에서 오른쪽 피연산자로 나눈 나머지를 반환한다. (모듈러) |

사용법은....그냥 수식처럼 쓰면 된다. 작성하면 해당 수식의 결과가 데이터로 만들어진다.
이 데이터는 다시 변수 등에 할당이 가능하다.

```java
int plus = 10 + 20;      // 30
int minus = 20 - 10;     // 10
int multiply = 20 * 10;  // 200
int divide = 20 / 10;    // 2
int modulo = 15 % 10;    // 5
```

연산의 우선순위는 일반적인 사칙연산과 동일하다. `%`의 경우 곱셈, 나눗셈과 동일한 우선순위를 가진다.

```java
// 괄호, 곰셈 나눗셈 모듈러, 그후 덧셈
int result = (10 + 5 % 3) / 3 + 2 * 3;  
```

기억할 것이, 정수와 정수를 나눌 경우, 결과는 정수로 나오게 된다. 이게 무슨 말이냐면, 일반적인 계산에서 
`3 / 2`를 할 경우 `1.5`가 되지만, Java에서는 결과가 정수로, 정확히는 몫이 나와서 `1`이 된다.
이는 결과를 할당할 변수를 `double`로 해도 마찬가지이다. 계산을 한 뒤 결과를 할당하기 때문.

```java
int notDivided = 21 / 10;
System.out.println(notDivided);  // 2

double doubleDivide = 21 / 10;
System.out.println(doubleDivide);  // 2.0
```

만약 실수를 이용해 연산을 한 뒤 정수 변수에 할당하려고 한다면 오류가 발생한다.

```java
int intResult = 10.1 + 2;  // error
```

### 형변환

위와 같은 상황에서는 데이터를 특정 정수형으로 변환해주고 싶을 수 있다.
이때 **형변환(Type Casting)**을 진행한다. 괄호를 이용해 해당 데이터를 바꾸고 싶은 자료형으로 바꿔준다.

```java
// <자료형> <변수명> = (<자료명>) <변환하려는 값>;
int integer = (int) 2.1;
```

상황에 따라 자료형이 자동으로 변환되는 경우도 있다. 예를 들어, 연산 결과가 `int`인 경우, 별다른 조작 없이 `long` 변수에 할당이 가능하다.
이를 묵시적(implicit) 형변환이라 한다. 아래 순서대로 묵시적 형변환이 가능하다.

```java
byte → short → int → long → float → double
```


```java
byte byteVar = 1;
System.out.println(byteVar);
short shortVar = byteVar;
System.out.println(shortVar);
int intVar = shortVar;
System.out.println(intVar);
```

반대 방향으로는 앞서 살펴본것 처럼 괄호를 이용해 형변환을 수동으로 해줘야 한다. 이를 명시적(explicit) 형변환이라 한다.

```java
double doubleVar = 10.24;
int castedDouble = (int) doubleVar;

System.out.println(castedDouble);
```

기준은 데이터의 표현 범위에 있다. 기본적으로 더 넓은 범위의 자료형에 더 좁은 범위의 데이터를 넣는건 암시적으로 동작하고, 반대는 명시적으로 해줘야 한다.
더 좁은 범위에 더 큰 값을 넣게될 수 있기 때문에, 데이터 손실이 일어날 수 있다.

```java
int dataLost = (int) 4294967296L;
System.out.println(dataLost);  // 0
```

> 위 예시의 결과가 `0`인 이유는 컴퓨터가 데이터를 저장하는 방법, 즉 비트와 연관이 있다.

`정수 / 정수`의 상황에서 결과를 실수로 받고싶을 때 명시적 형변환을 활용할 수 있다. 계산이 이뤄지기 전, 피연산자 중 하나를 실수 자료형으로 변형한다면
계산의 결과도 해당 실수 자료형으로 나오게 되므로, 정수와 정수를 나눌 때 실수 결과를 만들 수 있다.

```java
double doubleDivide = (double) 21 / 10;
System.out.println(doubleDivide);
```

참고로 `char` 자료형도 `int`를 비롯한 자료형으로 연산할 수 있다. 이 경우 일반적으로 해당 문자에 해당하는 유니코드로 변환된다고 보면 된다. `int` 자료형도
`char`로 변환이 되며, 유니코드를 따라 특정 문자로 변환된다. 물론 데이터 손실도 발생한다. 문자의 인코딩 디코딩에 관한 내용은 미래에 다시 작성하는걸로(...)

```java
int intPlusChar = 10;
char char1 = 65;
char char2 = 10;
System.out.println(intPlusChar + char1);           // 75
System.out.println((char) (intPlusChar + char1));  // K
System.out.println(char1 + char2);                 // 75
```

> 대부분의 경우 `char`을 정수로, 다시 `char`로 변환할때 사용하는 범위는 ASCII 코드의 범위에서
> 사용하게 된다. 그래서 대부분의 레퍼런스는 ASCII 코드를 이야기한다. 근데 사실 Java의 `char`은 2바이트로 UTF-16를 이용한다. 

몇몇 알고리즘 문제에서 이게 활용되는 경우가 많아서 간단히 설명하자면, `'A'`는 정수로 바꾸면 `65`가 되며, `'B'`는 `66`, `'C'`는 `67`...식으로 변환된다.
소문자의 경우도 비슷하다. `'a'`는 `97`, `'b'`는 `98`...

### 문자열 덧셈

문자열은 덧셈 연산자를 사용할 수 있다. 이 경우 두 문자열을 연결한 새로운 문자열이 만들어진다. 이는 가장 간단하게 문자열을 연결하는 방법...이긴 한데 나중에는
성능상의 단점으로 사용이 조금 지양된다. 특히 알고리즘 문제풀이와 같은 상황에서.

```java
String hello = "Hello,";
String space = " ";
String world = "World!";
System.out.println(hello + space + world);  // Hello, World!
hello += space + world;
System.out.println(hello);                  // Hello, World!
```

만약 문자열 덧셈 중 다른 자료형의 데이터가 포함된다면, 해당 데이터를 문자열로 변환한다. 형변환은 아니고 그냥 그렇게 동작한다.

```java
int date = 4;
// Today is 4th of July
System.out.println("Today is " + date + "th of July");
```

### 증감 연산자

특정 정수 변수의 데이터를 1씩 증가하거나 감소시키는 연산자다. 피연산자가 하나이기 때문에 단항 연산자라고도 부른다.
어떤 작업을 몇번 반복했는지 세는데 용이하다.

| 연산자  | 설명                |
|------|-------------------|
| `++` | 피연산자의 값을 1 증가시킨다. |
| `--` | 피연산자의 값을 1 감소시킨다. |

```java
int count = 0;
count++;
System.out.println(count);  // 1
count++;
System.out.println(count);  // 2
count--;
System.out.println(count);  // 1
```

증감 연산자는 변수의 앞뒤에 붙일 수 있는데, 동작의 차이가 있다.

- 변수의 앞에 붙을 경우, 코드가 실행되기 전 증감이 진행된다.
- 변수의 뒤에 붙을 경우, 코드가 실행되고 나서 증감이 진행된다.

```java
int count = 10;
System.out.println(++count);  // 출력: 11 (출력 전에 1 증가, 값: 11)
System.out.println(count++);  // 출력: 11 (출력 후에 1 증가, 값: 12)
System.out.println(--count);  // 출력: 11 (출력 전에 1 감소, 값: 11)
System.out.println(count--);  // 출력: 11 (출력 후에 1 감소, 값: 10)
```

### 할당 연산자

값을 할당하는데 사용하는 연산자를 부르는 말이다. `=`도 할당 연산자의 일종이다. 여기에 다른 산술 연산자랑 조합하면, 현재 변수에 할당된 값에, 오른쪽 피연산자(값)을 
왼쪽 피연산자(변수)에 계산한 뒤 할당할 수 있다. 이를 복합 할당 연산자라 부른다.

| 연산자  | 설명                                          | 사용         | 비교            |
|------|---------------------------------------------|------------|---------------|
| `=`  | 왼쪽의 피연산자에 오른쪽 피연산자를 할당한다.                   |            |               |
| `+=` | 왼쪽의 피연산자에서 오른쪽 피연산자를 더한 값을 왼쪽 피연산자에 할당한다.   | `a += 10;` | `a = a + 10;` |
| `-=` | 왼쪽의 피연산자에서 오른쪽 피연산자를 뺀 값을 왼쪽 피연산자에 할당한다.    | `a -= 10;` | `a = a - 10;` |
| `*=` | 왼쪽의 피연산자에서 오른쪽 피연산자를 곱한 값을 왼쪽 피연산자에 할당한다.   | `a *= 10;` | `a = a * 10;` |
| `/=` | 왼쪽의 피연산자에서 오른쪽 피연산자를 나눈 값을 왼쪽 피연산자에 할당한다.   | `a /= 10;` | `a = a / 10;` |
| `%=` | 왼쪽의 피연산자에서 오른쪽 피연산자로 나눈 나머지를 왼쪽 피연산자에 할당한다. | `a %= 10;` | `a = a % 10;` |

```java
int addTo = 1;   // 1
addTo += 10;     // 11
addTo *= addTo;  // 121
```

## 비교 연산자

두 피연산자를 비교하여 `boolean`을 반환한다. 특정 상황에서 코드를 실행시키는 제어문과 함께 사용하는 경우가 많다.

| 연산자  | 설명                                             |
|------|------------------------------------------------|
| `==` | 왼쪽의 피연산자와 오른쪽 피연산자가 같으면 참, 다르면 거짓을 반환한다.       |
| `!=` | 왼쪽의 피연산자와 오른쪽 피연산자가 다르면 참, 같으면 거짓을 반환한다.       |
| `>`  | 왼쪽의 피연산자가 오른쪽 피연산자 보다 크면 참, 작거나 같으면 거짓을 반환한다.  |
| `>=` | 왼쪽의 피연산자가 오른쪽 피연산자 보다 크거나 같으면 참, 작으면 거짓을 반환한다. |
| `<`  | 왼쪽의 피연산자가 오른쪽 피연산자 보다 작으면 참, 크거나 같으면 거짓을 반환한다. |
| `<=` | 왼쪽의 피연산자가 오른쪽 피연산자 보다 작거나 같으면 참, 크면 거짓을 반환한다.  |

```java
int small1 = 10;
int small2 = 10;
int big = 21;

System.out.println(small1 == small2);  // true
System.out.println(small1 == big);     // false
System.out.println(small1 != small2);  // false
System.out.println(small1 != big);     // true
System.out.println(small1 > big);      // false
System.out.println(small1 >= big);     // false
System.out.println(small1 >= small2);  // true
System.out.println(small1 < big);      // true
System.out.println(small1 <= big);     // true
System.out.println(small1 <= small2);  // true
```

### 논리 연산자

두개의 참 / 거짓 값에 대하여, 여러가지 조건을 확인하기 위해 사용하는 연산자이다. 모든 조건을 만족하는지, 하나의 조건이라도 만족하는지, 조건을 만족하지 않는지
등의 상황을 판단하기 위해 사용한다.

| 연산자   | 설명                                             |
|-------|------------------------------------------------|
| `&&`  | 왼쪽의 피연산자와 오른쪽 피연산자가 모두 참일 때 참, 아니면 거짓을 반환한다.   |
| `||` | 왼쪽의 피연산자와 오른쪽 피연산자 중 하나라도 참일 때 참, 아니면 거짓을 반환한다. |
| `!`   | 피연산자가 참이면, 거짓, 거짓이면 참을 반환한다. (부정)              |

```java
int temperature = 38;
int age = 20;

boolean notSick = temperature <= 37;    // 체온이 37도 이하
boolean isOfAge = age >= 20;            // 나이가 20살 이상
boolean canEnter = notSick && isOfAge;  // 입장 가능한가?
System.out.println(canEnter);

boolean isSunday = true;                  // 일요일인가?
boolean isHoliday = false;                // 공휴일인가?
boolean canRest = isSunday || isHoliday;  // 오늘은 쉬는날인가?
System.out.println(canRest);
```

복수의 논리 연산자가 조합되면, `!` → `&&` → `||` 의 순서대로 결과가 나온다.

```java
boolean a = true;
boolean b = false;
boolean c = false;

System.out.println(a && b || !c);      // true
System.out.println(a && b || a && c);  // false
System.out.println(c || b && a || c);  // false
```

## 비트 연산자

컴퓨터는 데이터를 비트 단위로 다룬다. 이는 Java도 마찬가지이다. 그리고 많은 프로그래밍 언어가 비트를 바탕으로 데이터를 비교, 계산하는 방법을 제공한다. 이를
비트 연산이라 한다. 정수를 기준으로 생각했을 때, 이진수로 변환하면 비트를 구할 수 있다. `11`과 `5`를 예로 들어보자.

```java
11 -> 1011
05 -> 0101
```

이후 두 숫자의 각 자리 값을 비교하거나, 비트를 이동시켜 새로운 값을 구하는게 비트 연산이다.

비트 연산은 응용 소프트웨어 개발 단계에서는 높은 추상화로 인해 많이 활용하지는 않지만, 몇몇 알고리즘 구현에 활용된다.

| 연산자   | 비트 연산                                                       |
|-------|-------------------------------------------------------------|
| `&`   | 두 수 각 자리의 비트가 둘다 1이면 결과 자리의 비트도 1 (AND 연산)                  |
| `|`   | 두 수 각 자리의 비트가 둘중 하나라도 1이면 결과 자리의 비트도 1 (OR 연산) |
| `^`   | 두 수 각 자리의 비트가 서로 다르면 결과 자리의 비트가 1 (XOR 연산)                  |
| `~`   | 각 자리의 비트를 1이면 0으로, 0이면 1로 (NOT 연산)                          |
| `<<`  | 오른쪽 피연산자의 수만큼 비트를 왼쪽으로 이동 (LEFT SHIFT 연산)                   |
| `>>`  | 오른쪽 피연산자의 수만큼 비트를 오른쪽으로 이동, 부호가 변하지 않는다. (RIGHT SHIFT 연산)   |
| `>>>` | 오른쪽 피연산자의 수만큼 비트를 오른쪽으로 이동, 새로운 비트는 0이 된다. (RIGHT SHIFT 연산) |

### AND 연산 `&`

각 자리의 비트를 비교했을 때, 둘다 `1`일 경우 결과의 해당 자리도 `1`이다. 하나라도 `0`일 경우 해당 자리는 `0`이 된다.

```java
/*
    1011 = 11
AND 0101 = 5
--------
    0001 = 1
*/
System.out.println(11 & 5);  // 1
```

### OR 연산 `|`

각 자리의 비트를 비교했을 때, 하나라도 `1`일 경우 결과의 해당 자리도 `1`이다. 둘다 `0`일 경우 해당 자리는 `0`이 된다.

```java
/*
    1011 = 11
OR  0101 = 5
--------
    1111 = 15
*/
System.out.println(11 | 5);  // 15
```

### XOR 연산 '^'

각 자리의 비트를 비교해서 둘의 자리가 다를 경우 결과의 자리가 `1`이 된다. 같을 경우 `0`이 된다. 베타적(Exclusive) OR 연산이다.

```java
/*
    1011 = 11
XOR 0101 = 5
--------
    1110 = 14
*/
System.out.println(11 ^ 5);  // 14
```

### NOT 연산 `~`

각 자리의 비트를 바꾼다. `0`을 `1`로, `1`을 `0`으로.

```java
/*
NOT 0000 1011 = 1111 0100 = -12
NOT 0000 0101 = 1111 1010 = -6
*/
System.out.println(~11);  // -12
System.out.println(~5);   // -6
```

> 왜 양수에서 음수가 되는지는 2의 보수의 내용을 찾아봐야 한다. 이도 미래에 다뤄보자...

### SHIFT 연산 `<<`, `>>`, `>>>`

하나의 숫자의 비트를 오른쪽 피연산자 만큼 이동한다. 이를 방향에 맞춰 왼쪽 또는 오른쪽으로 이동한다.

```java
/*
0000 1011 SHIFT LEFT  2 = 0010 1100 = 44
0000 1011 SHIFT RIGHT 2 = 0000 0010 = 2
*/
System.out.println(11 << 2);  // 44
System.out.println(11 >> 2);  // 2
```

SHIFT RIGHT 연산의 경우 오른쪽으로 이동하게 될 경우 왼쪽 끝의 비트가 어떻게 되는지에 따라 결과가 매우 달라지기 때문에(2의 보수), `>>`, `>>>`의 두가지
연산이 존재한다. `>>`의 경우 이전의 왼쪽 끝 비트를 보존하고, `>>>`의 경우 `0`이 된다.

```java
/*
>> 와 >>> 의 차이 예시
1111 1010 >> 2  = 1111 1110
1111 1010 >>> 2 = 0011 1110 
실제 연산시에는 더 많은 비트를 사용한다.
*/
System.out.println(-6 >> 2);   // -2
System.out.println(-6 >>> 2);  // 1073741822
```

## 삼항 연산자

세개의 피연산자를 가진다는 의미의 삼항 연산자이다. 첫 피연산자에는 참 또는 거짓으로 판별되는 값이, 뒤의 두 피연산자는 해당 값이 참 또는 거짓인지에 따라
어떤 결과일지를 판단하는 값을 작성한다.

```java
int temperature = 37;
String message = temperature < 38 ? "OK" : "Feverish";
```

이렇게 작성하게 될 경우, `temperature`의 값이 `38` 미만일 경우 `"OK"`가, 이상일 경우 `"Feverish"`가 `message에` 할당된다.
조건문 대신 값을 할당하는 용도로 사용할 수 있긴 한데, 너무 복잡한 활용은 코드의 가독성을 많이 떨어뜨리기 때문에 주의해야 한다.

```java
// x가 양수일때 y가 양수면 1, 아니면 2
// x가 음수일때 y가 양수면 3, 아니면 4
int result = (x > 0) ? ((y > 0) ? 1 : 2) : ((y > 0) ? 3 : 4);
```
